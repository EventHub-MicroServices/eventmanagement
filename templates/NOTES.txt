1. Get the status of your deployment:
  kubectl get deployments -n {{ .Values.global.namespace }}

2. Check all running pods:
  kubectl get pods -n {{ .Values.global.namespace }}

3. View service endpoints:
  kubectl get svc -n {{ .Values.global.namespace }}

4. View logs for a service (example: admin-service):
  kubectl logs -f deployment/admin-deployment -n {{ .Values.global.namespace }}

5. Access frontend (if LoadBalancer):
  FRONTEND_IP=$(kubectl get svc frontend-service -n {{ .Values.global.namespace }} -o jsonpath='{.status.loadBalancer.ingress[0].ip}')
  echo "Frontend available at: http://${FRONTEND_IP}:{{ .Values.frontend.service.port }}"

6. Database connection information:
   - PostgreSQL: postgres-db.{{ .Values.global.namespace }}.svc.cluster.local:{{ .Values.postgres.port }}
   - MongoDB: events-mongodb.{{ .Values.global.namespace }}.svc.cluster.local:{{ .Values.mongodb.port }}

7. For API Gateway access:
  kubectl get gateway -n {{ .Values.gateway.namespace }}

8. To upgrade your release:
  helm upgrade event-management . -f values.yaml

9. To uninstall:
  helm uninstall event-management

For more information and advanced configuration, see the README.md file.
